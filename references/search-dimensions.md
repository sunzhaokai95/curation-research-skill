# 多维资料搜索逻辑体系

本文件解决一个问题:一个主题到底要从哪些方向搜,才不会只得到几句定义和零散新闻。

搜索不是自由发挥。正式调研必须先运行:

```bash
python3 "<skill>/scripts/research_plan.py" --topic "主题" --type "A/B/C/D" --region "地域" --out "01_检索计划/research_plan.json"
```

`research_plan.json` 是后续搜索、证据卡和覆盖审计的依据。

先确认 `theme_profile.content_subject`。如果输入是“某某文化馆 / 某某博物馆 / 某某企业展厅 / 某某主题馆”,搜索主体资料时优先搜“某某”这个内容本体;只有机构、空间、项目建设、来源索引等方向才补搜完整项目名。否则会把主题本身的生平、历史、分类、工具、文献漏掉。

主题馆、兴趣社群、生活方式主题还要看 `theme_profile.subject_aliases`。遇到俗称、社群称谓、网络热词时,必须同时搜本体词和俗称;例如社群称谓只负责找到人群语言,本体词负责找到历史、工具、规则、分类、材料和行业资料。

## 搜索产物层级

搜索过程必须产生四层材料:

1. `search_results.jsonl`:真实搜索结果或查询任务。
2. `sources.jsonl`:可访问来源的标题、URL、发布日期、正文摘录。
3. `evidence_cards.jsonl`:从来源中抽出的可核验证据。
4. `coverage_audit.json`:必查核心是否满足证据覆盖。

只有第 3 层证据卡能进入导图。查询任务、网页摘要、模型印象都不能直接当导图事实。

## 五轮搜索法

每个重要方向都按五轮执行:

- **基础轮**:找概览、定义、关键词、同义词、权威入口。
- **权威轮**:抓官网、政府/监管、学术、年报、白皮书、标准、数据库、主流媒体、法院/专利/招标/交易所等。
- **下钻轮**:把重要人名、术语、型号、事件、法规、地名、作品、器物、材料、工艺、组织再单独搜索。
- **最新轮**:查当前年份、近一年、近 90 天;现代企业、政策、科技、产业、人物、消费主题必做。
- **交叉核验轮**:数字、融资、估值、事故、争议、历史年代、来源冲突至少找两个来源;单一来源必须标注。

五轮都要反映到证据卡里,不是只在脑中想过。

## 来源等级硬清单

### 企业品牌 / 产业科技

优先来源:

- 官网、新闻室、投资者关系页、产品文档、领导层页面。
- 交易所、监管文件、招股书、年报、法院公告、专利、招标、政府采购。
- 行业协会、咨询报告、主流财经媒体、产业数据库。
- 近一年、近 90 天、近 30 天新闻和监管进展。

硬性搜索:

- `<主题> IPO`
- `<主题> initial public offering`
- `<主题> funding valuation investors`
- `<主题> revenue contract backlog`
- `site:sec.gov <主题>`
- `<主题> lawsuit regulator license`
- `<主题> latest news <当前年份>`

- 文物图录、书画图录、考古/调查报告、学术论文、专著、教材。
- 当代保护、出版整理、数字化公开、学术会议、教育活动和纪念活动动态。

## 资料搜索雷达

每个维度都要问同一个问题:这个主题在该方向上有什么定义、事实、数据、人物、时间线、来源和争议?

### 1. 主题边界 / 基础定义

必须产出:

- 中文名、英文名、简称、别名、旧称。
- 这是什么,不是什么。
- 所属领域、上位概念、下位概念。
- 与相邻概念的区别。
- 核心术语表。

证据卡字段重点:定义、边界、别名、上下位概念、误区。

### 2. 时间历史 / 发展阶段

必须产出:

- 起源:最早何时出现、谁提出、在哪出现。
- 历史分期:每阶段时间、背景、特征、关键变化。
- 大事记:精确到年/月/日,写清人物、地点、事件、结果。
- 现状和已公开未来计划。

证据卡字段重点:时间、地点、人物、事件、结果、来源。

### 3. 地理空间 / 区域差异

必须产出:

- 发源地、核心地区、传播路线、分布格局。
- 国内外差异、区域政策差异、地域风格差异。
- 重要地点、设施、基地、场馆、遗址、路线、网络节点。
- 自然环境、资源条件、交通条件、生态背景。

### 4. 人物 / 群体 / 组织网络

必须产出:

- 创始人、代表人物、关键团队、管理层、专家、传承人、研究者、监管者。
- 用户、爱好者、从业者、受影响群体、社群结构。
- 组织机构、协会、企业、学校、实验室、平台、供应商、客户。
- 每个关键人物继续下钻:生平、任职、贡献、作品、公开讲话、争议、关联事件。

历史人物主题中,“生平”不是人物分支的一个小点,而是全图核心主干。

### 5. 分类谱系 / 类型系统

必须产出:

- 分类依据。
- 每个子类是什么,不是什么。
- 每类的组成、原理、场景、代表对象、数据和来源。
- 不同分类体系之间的差异。

禁止只列类型名。

### 6. 关键对象 / 载体 / 实物 / 作品 / 档案

必须产出:

- 器物、工具、材料、设备、产品、作品、文本、档案、标本、藏品、文物、影像、数据集。
- 名称、年代、材质、尺寸、型号、作者、来源、收藏/保存机构、公开编号。
- 对象的发现、制作、流转、使用、修复、出版、版本差异。

非物质主题也必须找物质载体、过程载体或影像/文本载体。

### 7. 原理 / 机制 / 工艺 / 操作流程

必须产出:

- 基本原理、运行机制、系统架构、工艺流程、操作步骤。
- 材料、工具、设备、软件、算法、标准、参数。
- 技术难点、限制条件、失败原因、迭代路径。
- 生活方式或群体行为主题要拆行为流程、入门路径、熟练路径、日常实践。

### 8. 场景 / 行为 / 使用方式 / 日常实践

必须产出:

- 典型场景、时间节律、空间条件、参与者角色。
- 入门者怎么接触,熟练者怎么进阶。
- 常见动作、流程、规则、仪式、训练方式。
- 使用限制、风险、误区、常见失败。

### 9. 数据 / 规模 / 市场 / 消费

必须产出:

- 人群规模、用户数、游客量、产量、销量、装机量、出货量、订单量、市场规模。
- 价格带、消费层级、成本结构、收入模式、渠道。
- 产业链:上游供给 → 中游生产/平台 → 下游用户/场景。
- 数据口径、统计年份、发布机构。

没有年份、单位、口径和来源的数字不能当确定资料。

### 10. 企业 / 品牌 / 机构公开资料

涉及企业、机构、品牌时必查。若主题不是企业,本方向转译为机构网络、收藏机构、科研机构、主管机构、社群组织、任务网络。

必须产出:

- 官网栏目、主体信息、使命/宗旨、组织结构。
- 产品线、业务线、客户、合作伙伴、子公司、品牌矩阵。
- 历史文化类:纪念馆、博物馆、图书馆、档案馆、研究会、主管部门、出版机构、保护单位。
- 科学教育类:国际组织、科研机构、大学、天文台/实验室、任务团队、数据中心、标准组织。

### 11. 财经资本 / 交易 / 最新商业动态

现代企业、产业、平台、消费主题必查,防止漏掉上市、融资、估值和最新变化。

必须产出:

- 上市/IPO 状态、交易所、股票代码、招股书、撤回/延迟/否认、传闻状态。
- 融资轮次:时间、金额、估值、投资方、资金用途。
- 股权/投资人:创始人持股、员工期权、主要投资人、二级市场交易。
- 财务数据:营收、利润/亏损、现金流、债务、订单、合同、客户数。
- 重大交易:并购、剥离、采购、政府合同、战略合作。
- 近一年、近 90 天、近 30 天新闻与监管进展。

非商业主题转译为当代保护、出版、数字化、研究进展、项目建设、公共传播、行业动态。

### 12. 政策 / 法规 / 制度 / 监管 / 伦理

必须产出:

- 法律法规、行业标准、管理办法、许可制度、禁令。
- 监管机构、行业协会、认证制度、处罚和合规要求。
- 文物保护、非遗保护、生态保护、知识产权、数据安全、伦理争议。
- 政策对主题的影响:推动、限制、转型、风险。

### 13. 社会文化 / 符号 / 语言 / 传播

必须产出:

- 象征意义、价值观、审美、哲学、民俗、仪式、习俗。
- 圈层语言、黑话、流行语、网络梗、身份认同。
- 文学、艺术、影视、媒体、短视频、社交平台中的呈现。
- 公众舆论、传播节点、代际差异、地域差异。

文化解释必须回到来源,不能只写主观感受。

### 14. 故事 / 事件 / 争议 / 失败

必须产出:

- 起源故事、历史典故、关键事件、人物轶事。
- 标志性事件:时间、地点、人物、经过、结果、后续影响。
- 争议、事故、失败、纠纷:不同说法、证据来源、目前状态。
- 流传版本与权威版本差异。

这里搜事实故事,不是展项创意。

### 15. 学术 / 权威资料 / 文献出处

必须产出:

- 学术论文、专著、教材、行业白皮书、年报、统计年鉴、标准文件。
- 地方志、正史、碑刻、谱牒、报刊、书信、口述史、照片、影像档案。
- 专家访谈、公开演讲、课程资料、数据库。
- 作者、年代、卷次篇名、版本差异、现代整理本。

### 16. 同类对象 / 横向事实对照

只比事实,不写启示。

必须产出:

- 同类对象至少 3 个。
- 基本情况、时间线、规模、分类、数据、政策、资本状态。
- 相同点、差异点、资料来源。
- 相邻领域、替代对象、不同技术路线、不同商业模式。

### 17. 最新动态 / 当前状态

所有现代主题都单独做。传统历史文化主题也要做,但“最新”含义是最新研究、保护、出版、数字化和公共传播状态。

必须产出:

- 当前年份、近一年、近 90 天、必要时近 30 天。
- 新闻日期、事件发生日期、文件发布日期、政策生效日期。
- 官方确认、媒体报道、传闻、估算、待核实状态。
- 历史文化类:近年论文、图书出版、馆藏数据库更新、地方志整理、文物修复、保护政策、纪念活动。
- 科学教育类:任务发射/运行、数据发布、论文/预印本、命名公告、观测结果、科普机构更新。
- 生活方式类:平台趋势、活动赛事、产品变化、社群语言、消费数据、安全规范。

### 18. 来源索引 / 待核实清单

从搜索开始就整理来源。

必须产出:

- 官方来源、监管来源、学术来源、媒体来源、行业来源、用户资料。
- 每个重要事实对应来源。
- 单一来源、口径冲突、过期信息、待人工核实项。
- 未找到权威来源时,写清已查哪些来源和缺少什么证据。

## 认知阶梯搜索法

重要概念不能只搜定义。按以下顺序继续下钻:

1. 直观经验:`<概念> 入门`、`<概念> beginner`、`<概念> basics`。
2. 定义边界:`<概念> definition`、`<概念> 是什么`、`<概念> vs <相似概念>`。
3. 分类系统:`<概念> types`、`<概念> classification`、`<概念> 分类`。
4. 组成对象:`<概念> components`、`<概念> 组成 对象 材料 工具`。
5. 机制方法:`<概念> how it works`、`<概念> process`、`<概念> 方法 原理`。
6. 场景行为:`<概念> use cases`、`<概念> practice`、`<概念> 场景 行为`。
7. 数据证据:`<概念> data`、`<概念> statistics`、`<概念> 参数 数据`。
8. 历史前沿:`<概念> history`、`<概念> latest`、`<概念> <当前年份>`。
9. 误区争议:`<概念> misconceptions`、`<概念> controversy`、`<概念> 误区 争议`。

历史人物和文学艺术主题的“直观经验”可替换为公众熟悉入口、常见名句、常见作品、常见地名或常见误读,但不能停在那里。

- 搜索结果是否是真实 URL,而不是只有 query_task?
- `sources.jsonl` 是否有足够公开来源?
- `evidence_cards.jsonl` 是否通过校验?
- `coverage_audit.py` 是否通过?
- 是否覆盖至少 12 个资料方向?
- 重要概念是否再次作为小主题下钻?
- 是否有时间线、人物、分类、对象、机制、场景、数据、政策、故事、学术来源、最新动态?
- 是否有权威来源和交叉核验?
- 是否存在只有一句定义或几个短词的分支?
- 是否存在占位句或策展语言?
