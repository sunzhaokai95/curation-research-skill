# 多维资料搜索逻辑体系

本文件解决一个问题:一个主题到底要从哪些方向搜,才不会只得到几句定义、零散新闻或来源碎片。

正式调研必须先运行:

```bash
python3 "<skill>/scripts/research_plan.py" --topic "主题" --type "A/B/C/D" --region "地域" --out "01_检索计划/research_plan.json"
```

先确认 `theme_profile.content_subject`。如果输入是“某某文化馆 / 某某博物馆 / 某某企业展厅 / 某某主题馆”,搜索主体资料时优先搜“某某”这个内容本体;只有机构、空间、建设、运营或来源索引方向才补搜完整项目名。

## 搜索产物层级

搜索过程必须产生五层材料:

1. `search_results.jsonl`:真实搜索结果或查询任务。
2. `sources.jsonl`:普通公开网页文本,或人工/浏览器读取后的公开摘录。
3. `evidence_cards.jsonl`:从来源中抽出的可核验证据。
4. `coverage_audit.json`:必查核心是否满足证据覆盖。
5. `调研大纲.md`:知识系统化后的导图源文件。

证据卡能校准事实,但不能决定最终导图形状。最终导图要先按知识系统组织,再把来源、数据和争议放在可见子节点中。

## 来源层级

### 1. 知识入口层

优先查百度百科、维基百科、百科概览、专业词典、教材概览、入门手册、综述文章。用途是建立:

- 名称、别名、英文名、旧称。
- 上位概念、下位概念、相邻概念。
- 分类系统、术语表、常见误区。
- 代表人物、代表对象、历史阶段的初步索引。

百科不是最终答案,但必须作为知识地图入口。不能因为追求权威来源而放弃百科带来的系统结构。

### 2. 解释建模层

用模型知识把入口资料和证据卡组织成可读知识系统:

- 解释“是什么 / 不是什么”。
- 解释“为什么这样分类”。
- 解释工具、机制、流程、行为、术语差异。
- 把碎片来源合并成章节顺序和概念关系。

模型知识不能直接填具体年份、人物关系、地点、政策条款、市场数字、最新新闻、来源标题或 URL。

### 3. 硬事实校准层

硬事实必须查来源:

- 官方网站、机构页面、新闻室、产品文档。
- 学术论文、专著、教材、标准、白皮书。
- 主流媒体、行业协会、产业数据库、年报、招股书。
- 政府、监管、法院、交易所、官方数据库等权威来源。

注意:政府、监管、法院、交易所、官方数据库和 robots 禁止页面只允许浏览器/人工读取后用 `manual_source_note.py` 记录摘录,不得用脚本爬取。

### 4. 最新动态层

现代企业、产业、政策、科技、消费、平台、人物、社交传播主题必须查当前年份、近一年、近 90 天,必要时近 30 天。传统文化主题也要查最新研究、出版、保护、数字化、公共传播和教育活动。

## 五轮搜索法

每个重要方向都按五轮执行:

- **入口轮**:百科、词典、教材、入门手册,建立术语地图和知识骨架。
- **权威轮**:搜索官方、学术、监管、标准、数据库、主流媒体等;官方/监管等受限来源只人工读取记录。
- **下钻轮**:把人名、术语、型号、事件、法规、地名、作品、器物、材料、工艺、组织再单独搜索。
- **最新轮**:查当前年份、近一年、近 90 天。
- **交叉核验轮**:数字、融资、估值、事故、争议、历史年代、来源冲突至少找两个来源;单一来源必须标注。

如果搜索结果只回答了“是什么”,但没有给出时间线、分类、对象、流程、数据、组织、政策、来源或争议,这一轮搜索不算完成。

## 查询任务必须带目标

从 `research_plan.py` 或 `research_loop.py` 得到的查询,正式搜索时必须同时看:

- `research_goal`:这条查询要补哪一个必查核心,要解决什么缺口。
- `expected_evidence_fields`:搜索后必须抽取哪些事实字段。
- `stop_condition`:该核心达到覆盖审计阈值之前不能停止。
- `visited_urls`:已经用过的来源,不能重复当作新增证据。

## 资料搜索雷达

每个维度都要问同一个问题:这个主题在该方向上有什么定义、事实、数据、人物、时间线、来源和争议?

### 1. 主题边界 / 基础定义

中文名、英文名、简称、别名、旧称;这是什么,不是什么;所属领域、上位概念、下位概念;与相邻概念的区别;核心术语表。

### 2. 时间历史 / 发展阶段

起源、历史分期、大事记、现状和已公开未来计划。历史文化人物必须有生平年谱。

### 3. 地理空间 / 区域差异

发源地、核心地区、传播路线、分布格局、国内外差异、重要地点、设施、场馆、遗址、路线、自然环境和交通条件。

### 4. 人物 / 群体 / 组织网络

创始人、代表人物、专家、传承人、研究者、用户、爱好者、从业者、协会、企业、学校、平台、供应商、客户。关键人物必须继续下钻身份、贡献、事件、作品、争议和来源。

### 5. 分类谱系 / 类型系统

分类依据、每个子类是什么、每类的组成、原理、场景、代表对象、数据和来源。禁止只列类型名。

### 6. 关键对象 / 载体 / 实物 / 作品 / 档案

器物、工具、材料、设备、产品、作品、文本、档案、标本、藏品、文物、影像、数据集。非物质主题也必须找物质载体、过程载体或影像/文本载体。

### 7. 原理 / 机制 / 工艺 / 操作流程

基本原理、运行机制、系统架构、工艺流程、操作步骤、材料、工具、参数、限制条件、失败原因和迭代路径。

### 8. 场景 / 行为 / 使用方式 / 日常实践

典型场景、时间节律、空间条件、参与者角色、入门路径、进阶路径、常见动作、规则、安全限制、误区和常见失败。

### 9. 数据 / 规模 / 市场 / 消费

人群规模、用户数、游客量、产量、销量、市场规模、价格带、消费层级、成本结构、收入模式、渠道和产业链。数字必须有年份、单位、口径和来源。

### 10. 企业 / 品牌 / 机构公开资料

涉及企业、机构、品牌时必查。非企业主题转译为机构网络、收藏机构、科研机构、主管机构、社群组织或任务网络。

### 11. 财经资本 / 交易 / 最新商业动态

现代企业、产业、平台、消费主题必查,防止漏掉上市、融资、估值和最新变化。非商业主题转译为当代保护、出版、数字化、研究进展、项目建设、公共传播和行业动态。

### 12. 政策 / 法规 / 制度 / 监管 / 伦理

法律法规、行业标准、管理办法、许可制度、禁令、监管机构、行业协会、认证制度、处罚、合规要求、生态伦理和安全风险。官方法规页面人工记录。

### 13. 社会文化 / 符号 / 语言 / 传播

象征意义、价值观、审美、民俗、仪式、圈层语言、黑话、流行语、网络梗、身份认同、媒体呈现和公众舆论。文化解释必须回到来源和知识框架,不能只写主观感受。

### 14. 故事 / 事件 / 争议 / 失败

起源故事、历史典故、关键事件、人物轶事、争议、事故、失败、纠纷、不同说法和目前状态。这里只搜事实故事,不是展项创意。

### 15. 学术 / 权威资料 / 文献出处

学术论文、专著、教材、行业白皮书、年报、统计年鉴、标准文件、地方志、正史、碑刻、谱牒、报刊、书信、口述史、照片、影像档案。

### 16. 同类对象 / 横向事实对照

至少 3 个同类对象。只比事实:基本情况、时间线、规模、分类、数据、政策、资本状态、相同点、差异点和来源。

### 17. 最新动态 / 当前状态

所有现代主题单独做。传统历史文化主题的“最新”包括最新研究、保护、出版、数字化和公共传播状态。

### 18. 来源索引 / 待核实清单

从搜索开始整理来源。写清官方来源、监管来源、学术来源、媒体来源、行业来源、用户资料,以及单一来源、口径冲突、过期信息和待人工核实项。

## 认知阶梯搜索法

重要概念不能只搜定义。按以下顺序继续下钻:

1. 直观经验:`<概念> 入门`、`<概念> beginner`、`<概念> basics`。
2. 定义边界:`<概念> definition`、`<概念> 是什么`、`<概念> vs <相似概念>`。
3. 分类系统:`<概念> types`、`<概念> classification`、`<概念> 分类`。
4. 组成对象:`<概念> components`、`<概念> 组成 对象 材料 工具`。
5. 机制方法:`<概念> how it works`、`<概念> process`、`<概念> 方法 原理`。
6. 场景行为:`<概念> use cases`、`<概念> practice`、`<概念> 场景 行为`。
7. 数据证据:`<概念> data`、`<概念> statistics`、`<概念> 参数 数据`。
8. 历史前沿:`<概念> history`、`<概念> latest`、`<概念> <当前年份>`。
9. 误区争议:`<概念> misconceptions`、`<概念> controversy`、`<概念> 误区 争议`。

## 搜索完成自检

- 是否使用百科和入门资料建立知识骨架?
- 搜索结果是否是真实 URL,而不是只有 query_task?
- 官方/监管/政府/交易所/数据库来源是否人工记录,没有脚本抓取?
- `sources.jsonl` 是否有足够来源?
- `evidence_cards.jsonl` 是否通过校验?
- `coverage_audit.py` 是否通过?
- 是否覆盖至少 12 个资料方向?
- 重要概念是否再次作为小主题下钻?
- 是否有时间线、人物、分类、对象、机制、场景、数据、政策、故事、学术来源、最新动态?
- 是否存在只有一句定义或几个短词的分支?
- 是否存在占位句、字段腔、方法标签或策展语言?
