# 调研方法论总则

本 skill 只做一件事:为后续策展大纲准备一份**足够深、足够全、足够新、可核验、可直接打开的 XMind 资料导图**。用户明确要求报告、Word、docx 或完整交付时,可基于同一批证据卡追加资料报告型 Word。

默认最终成果不是调研报告,不是策展方案,不是展示建议。默认只交 `.xmind`。完整交付模式追加的 `.docx` 也是资料报告,不是策展方案。

## 生产链路

正式调研必须遵循“证据管线先于导图”:

1. 主题确认:类型、地域、时间范围。
2. 检索计划:`research_plan.py` 生成 `research_plan.json`。
3. 搜索收集:`search_collect.py` 搜索或导出查询任务。
4. 来源抓取:`fetch_sources.py` 抽取公开页面文本。
5. 证据卡:从来源中抽取 `evidence_cards.jsonl`。
6. 证据校验:`evidence_cards.py validate`。
7. 覆盖审计:`coverage_audit.py`。
8. 导图大纲:覆盖通过后才写 `调研大纲.md`。
9. 大纲审计:`outline_audit.py`。
10. 生成 XMind:`md_to_xmind.py`。
11. 最终审计:`xmind_audit.py`。
12. 完整交付模式:报告 Markdown:`report_from_evidence.py`。
13. 报告审计:`report_audit.py`。
14. 生成 Word:`report_to_docx.py`。

任何一步失败,都回到上一步补资料,不要用占位句糊过去。

## 资料价值观

资料调研的价值不在于几句观点,而在于保留后续大纲可调用的事实材料:

- 定义、边界、上下位概念。
- 历史阶段、时间线、事件过程。
- 人物、组织、群体、关系网络。
- 地理空间、区域差异、场景条件。
- 分类谱系、对象、器物、作品、文本、档案。
- 原理、流程、工艺、操作方法。
- 数据、规模、价格、市场、资本、政策。
- 社会文化、语言符号、媒介传播。
- 学术文献、权威来源、争议和待核实项。

每条重要资料都要尽量回答:谁、何时、何地、发生什么、数据多少、来源在哪、来源可信度如何、是否有不同说法。

## 从后续大纲反推调研深度

完整策展大纲会展开到:

- 篇章 / 单元。
- 组团 / 小节。
- 具体知识点。
- 具体人物、事件、器物、地点、作品、工艺、政策、数据。
- 说明文字、资料来源、图片/实物/档案线索。

因此前期资料导图必须比大纲更细。它不能只有“主题-分类-几个关键词”,而要把每个概念拆到能直接支撑正文写作。

反推检查必须落到节点:

- 每个一级资料分支至少能反推出 2-3 个后续大纲小节。
- 每个重要概念至少能支撑一段 150-300 字正文。
- 每个资料组至少包含对象、事件、人物、时间、地点、数据、来源中的 3 类。
- 如果一个分支只能写出口号、情绪判断或空泛概括,说明资料密度不足。

## Word 资料报告的定位

Word 报告不是把导图节点复制成文档,而是把同一批证据卡转成可阅读、可追溯、层级清楚的资料文本。它参考策展大纲的结构感,但不进入策展判断。

正确的 Word 报告应当具备:

- 总述:说明主题边界、资料范围、来源基础和阅读方式。
- 章节:每章回答一个资料问题,如定义、历史、人物、对象、机制、空间、数据、监管、同类对照。
- 段落:把多条证据卡合并成连续说明,句子有长短变化,不是短句清单。
- 来源口径:说明哪些资料来自官方、监管、学术、媒体、数据库或估算。
- 待核实问题:明确哪些数据会变化,哪些来源需要后续跟进。

Word 报告不得出现:

- 展项建议、展示形式、空间表达、观众体验、策展主题。
- 证据卡字段名堆叠,如“来源类型为”“可信度标注为”。
- “需要补”“正式使用时要”等内部任务句。
- 只有标题和短句,没有连续段落。

## 内容丰富不是分支越多

失败的大纲常见两种:

- 分支很多,但每个分支只有短词或一句话。
- 分支很多,但内容是“需要补”“正式使用时要”的内部待办。
- 分支很多,但内容是“继续下钻方向”“关联资料面”“下钻方式”的方法标签,没有真正增加事实。

正确标准是:

- 每个重要概念都有多层解释。
- 每个关键事实能追到来源。
- 每个分类名词都有定义、边界、典型特征、使用/出现条件、来源。
- 每个数字有年份、单位、口径和来源。
- 每个人物有身份、关系、事件、时间、来源。
- 每个作品/器物/档案有名称、年代、作者/相关人物、内容/形制、流传/收藏、来源。
- 每个“来源”节点都写成具体来源名、URL、发布时间、来源性质和可采信范围,不要写“来源类型为...”这类证据卡字段提示。

## 证据卡要求

证据卡是导图的原料,不是报告。每条证据卡至少包含:

- `claim`:完整事实陈述,不能只写短词。
- `core`:对应 `research_plan.json` 的必查核心。
- `dimension`:对应搜索维度。
- `time`:时间或年代。
- `people`:人物。
- `places`:地点。
- `objects`:作品、器物、事件、机构、产品、政策等对象。
- `data`:数字、金额、规模、参数、口径。
- `source_title`:来源标题。
- `source_url`:来源 URL 或用户资料文件名。
- `source_type`:government / academic / institution_or_database / media / web 等。
- `confidence`:官方确认 / 监管文件 / 学术资料 / 机构资料 / 媒体报道 / 行业资料 / 分析师估算 / 单一来源 / 待核实。

不能把 `evidence_seed.jsonl` 当证据卡。seed 只是待抽取材料。

每条证据卡至少要填实 `time`、`people`、`places`、`objects`、`data` 中的 2 个字段。字段为空、写“不详/待补/正式使用时要”都不能算有效证据。覆盖审计还会检查每个必查核心是否拥有足够事实字段,不是只数来源数量。

## 禁止越界

不要写:

- 展示形式、展项创意、互动方式、空间表达、展线设计。
- 策展主题、核心张力、独特价值、主题口号。
- 同类展馆可借鉴点、展陈启示、观众体验建议。

要写:

- 资料事实是什么。
- 来源在哪里。
- 发生在什么时候。
- 涉及谁。
- 数据是多少。
- 谁这么说。
- 是否官方确认。
- 还有哪些不同说法。

## 先破题

开搜前必须做:

1. 地域范围:国内中国 / 国际 / 特定地区。
2. 时间范围:历史范围 + 最新资料截止日期。
3. 主题切分:正确领域,避免误搜。
4. 主题本体:先剥离“馆/展厅/主题空间/项目名称”等外壳,找出真正要研究的内容对象。
5. 名称体系:中文名、英文名、简称、旧称、别名、法人名、术语名、代码名。
6. 上下位关系:母领域、子领域、相邻主题。
7. 易混淆项:同名对象、相似概念、网络热词歧义。
8. 权威来源路线:官网、政府/监管、学术、主流媒体、标准、法院、专利、招标、交易所、数据库等。

这些内容必须成为导图第一分支「主题解读」。

## 类型硬核

### A 企业 / 机构 / 品牌

必须查:

- 主体身份、法人/组织信息、总部、成立时间、创始人。
- 领导层、组织结构、子公司、品牌矩阵。
- 产品线、业务线、客户、合作伙伴、供应商。
- 技术能力、生产/交付体系、关键设施。
- 市场规模、产业链、商业模式、价格带。
- 上市/IPO、融资、估值、股权、收入、订单、合同。
- 监管、许可、诉讼、处罚、合规风险。
- 近一年、近 90 天、必要时近 30 天动态。

### B 博物馆 / 文化馆 / 历史文化

必须查:

- 主题定义、别名、地域范围、历史分期。
- 生平年谱或对象年表。
- 家族、师友、同僚、社群、传承人、研究者。
- 时代制度背景。
- 地理轨迹、遗址、纪念地、空间遗存。
- 作品、文献、版本、注本、题跋、书信、档案。
- 藏品、文物、标本、照片、影像、非遗项目。
- 地方志、正史、碑刻、谱牒、报刊、论文、专著、图录。
- 当代保护、出版整理、数字化公开、纪念活动、研究动态。

### C 文旅 / 主题馆 / 主题空间

必须先查主题本体:

- 主题是什么,不是什么,与相邻主题区别。
- 主题历史、分期、传播、当代变化。
- 主题分类、对象、工具、材料、作品、文本、影像。
- 参与者、群体、行为流程、使用场景、入门路径、进阶路径。
- 文化符号、圈层语言、媒介传播、消费结构、争议误区。
- 政策、伦理、生态、安全、行业规范。
- 最新动态与趋势。

只有主题绑定具体目的地或运营项目时,才额外补资源本底、区位交通、客群市场、运营主体、投资建设、政策规划、生态承载。

## 搜新标准

现代企业、机构、政策、科技、产业、人物、消费、社交传播主题必须查:

- 当前年份。
- 近一年。
- 近 90 天。
- 必要时近 30 天。
- 监管、标准、法院、交易所、政府公告、招标、专利、官网新闻。

传统历史文化主题也必须查最新,但口径是:

- 新出版。
- 新校注。
- 新研究。
- 新馆藏公开。
- 新数字资源。
- 新保护政策。
- 新纪念活动。
- 新教育/公共传播项目。

时效信息必须写清发布时间、事件发生时间、来源状态和是否可能变化。

## 占位句污染判定

导图不是工作待办清单。以下内容不得出现在最终 `.xmind`:

- “正式使用时要关联时间、人物、地点、作品、来源或版本差异。”
- “需要继续绑定具体时间、人物、事件、文献、实物或空间坐标。”
- “需要在正式资料中补齐。”
- “需要用文集、注本、数据库和学术资料核验。”
- “发布时间/实施主体/数据口径/影响范围/来源可信度需要保留。”
- “可提供或需要补充。”
- “不能只作为短词存在。”
- “待补:”“待补充正式出处。”
- “已列入正式调研来源或待补来源清单。”

正确处理:

- 已有资料:直接写事实、数据、时间、地点、人物、来源。
- 暂无资料:写“未找到权威来源”,并列出已查来源和缺口对象。
- 不把内部检查表、搜索任务、质量提醒变成导图节点。

## 方法标签伪内容判定

导图不能把方法论骨架本身当成资料。以下词语如果进入最终 `.xmind`,说明只是把“怎么继续写”的提示搬进了结果,而不是完成调研:

- “继续下钻方向”
- “下钻方式”
- “关联资料面”
- “资料密度检查”
- “概念资料卡”
- “认知阶梯骨架”
- “这一层不是结论”
- “作为资料节点”
- “来源类型为”
- “可信度标注为”
- “对应证据卡”

正确处理:

- 不写“继续下钻方向:生平、作品、地理”,而是直接写“生平年谱”“作品系统”“地理轨迹”及其具体事实。
- 不写“关联资料面:人物、地点、来源”,而是直接写人物是谁、地点在哪、来源是什么。
- 不写“来源类型为 government”,而是写“美国证券交易委员会 SEC EDGAR:文件类型 S-1,提交日期...,URL...”。
- 不写“可信度标注为媒体报道”,而是写“路透社报道:发布时间...,信息来自匿名知情人士,需与监管文件区分”。

## 短叶子判定

允许短的结构功能节点:

- 定义、来源、链接、时间、经过、结果、争议、作用、用途、人物、机构、地点、数据。

不允许短的资料内容节点:

- 人物名、作品名、器物名、地名、术语名、分类名、年份、数字、机构名如果已经是末端,必须继续补身份、解释、来源、上下文或数据口径。

## 合规边界

- 不下载盗版书籍/论文,只给书单与正规获取渠道。
- 企业信息走公开渠道。
- 需登录的系统只给查询线索,不绕过访问控制。
- 对敏感技术、国家安全、个人隐私等内容只整理公开资料和制度边界。
