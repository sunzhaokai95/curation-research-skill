# 资料型思维导图母骨架与概念深挖范式

本 skill 的导图不是调研报告,不是策展方案,而是**资料库式 XMind**。

中心主题 = 调研主题。一级分支 = 内容资料维度。所有解释、事实、数据、来源都必须作为可见子节点出现。

用户明确要求 Word 报告时,Word 与 XMind 分工如下:

- XMind 保存资料颗粒度,每个概念继续下钻到事实、数据、来源和争议。
- Word 报告组织阅读顺序,把证据卡和导图资料转成章节化、段落化、可阅读文本。
- Word 不得替代 XMind 的深度,也不得脱离证据卡另写无来源报告。

## 导图写作前置条件

写 `调研大纲.md` 前必须已经完成:

- `research_plan.json`
- `search_results.jsonl` 或真实来源清单
- `sources.jsonl`
- `evidence_cards.jsonl`
- `evidence_cards.py validate` 通过
- `coverage_audit.py` 通过
- 如果覆盖审计曾失败,已经运行 `research_loop.py` 并完成补搜返工

没有证据卡,不得按骨架空写。骨架是组织资料的容器,不是生成内容的模板。

## 禁止内容

资料型导图中不要出现:

- 展项建议、展示形式、空间表达、展线、互动装置、展厅策划转译。
- 独特价值、核心张力、核心矛盾、主题口号、策展主题。
- 可借鉴点、展陈启示、观众体验设计。
- “建议做成...”“适合展示...”“可以作为展厅...”等方案语言。
- 真实项目名、客户名、测试案例名、私有文件路径。
- “正式使用时要...”“需要继续...”“需要补...”“不能只作为短词存在”等内部待办句。
- “继续下钻方向”“下钻方式”“关联资料面”“概念资料卡”“来源类型为”“可信度标注为”等方法标签。

如果脑中出现方案判断,改写成资料事实:

- 错误:`适合做成互动展项`
- 正确:`<事件/对象> 的发生时间、参与者、过程、结果、来源和争议`

## 呈现原则

- 默认不使用 `>` 备注;备注数目标为 0。
- 不使用 `概念  >  解释` 这种压缩写法。
- 不把多个解释塞在一个标题里。
- 每个末端资料节点必须是一条完整信息,不能是孤立短词。
- 能继续拆就继续拆,把信息放到下一层。
- 不用“继续下钻方向 / 关联资料面 / 下钻方式”来制造层级;层级必须由具体事实、分类、过程、数据、来源自然展开。
- 不用“时间口径为 / 相关人物包括 / 涉及对象包括 / 数据口径包括”这种字段腔;应转成“时间”“人物”“对象”“数据”等自然节点。
- 资料不足时只能有两种写法:继续补搜后写入事实;或写成明确缺口,例如“未找到权威来源:已查 A/B/C,尚缺某对象的时间/地点/来源”。

## 统一内容资料母骨架

A 企业/机构/品牌、B 博物馆/文化馆/历史文化、C 文旅/主题馆/主题空间、D 其他主题,都使用同一套母骨架。类型只改变加权和必查项,不要生成四套互相割裂的目录。

```markdown
# <主题>资料调研
- 主题解读
  - 地域范围
  - 时间范围
  - 主题本体
  - 主题切分
  - 名称体系
  - 上下位概念
  - 易混淆对象
  - 权威来源路线
  - 检索计划摘要
- 基础定义与主题边界
- 历史源流与时间线
- 地理空间与区域差异
- 人物群体与组织网络
- 分类谱系与子主题系统
- 关键对象 / 载体 / 实物 / 作品 / 档案
- 原理机制 / 工艺流程 / 操作方法
- 场景行为 / 使用方式 / 生活实践
- 数据规模 / 产业市场 / 消费结构
- 企业机构 / 品牌主体 / 组织网络
- 财经资本 / 当代动态 / 最新状态
- 政策法规 / 制度标准 / 监管伦理
- 社会文化 / 符号语言 / 媒介传播
- 故事事件 / 争议失败 / 版本差异
- 学术文献 / 权威资料 / 来源出处
- 最新动态与当前状态
- 同类对象事实对照
- 来源索引与待核实清单
```

## 分支适配规则

统一母骨架不是机械模板。一级分支承担的是资料功能,不是固定词语。遇到不适配主题时,允许改名,但必须保留同等资料功能。

适配步骤:

1. 先判断主题主类型和副类型。
2. 保留母骨架的资料功能:定义、历史、空间、人物、分类、对象、机制、场景、数据、机构、资本/当代动态、政策、文化、故事、文献、最新、同类、来源。
3. 把不适配的词转成该主题能成立的表达,不要直接删除。
4. 转译后的分支仍要下钻到事实、数据、来源和版本差异。

常见转译:

| 母骨架分支 | 主题不适配时的转译方向 |
| --- | --- |
| 企业机构 / 品牌主体 / 组织网络 | 研究机构 / 学术组织 / 主管机构 / 收藏机构 / 社群组织 / 任务网络 |
| 财经资本 / 当代动态 / 最新状态 | 当代保护 / 出版 / 数字化 / 研究进展 / 项目建设 / 公共传播 / 行业动态 |
| 数据规模 / 产业市场 / 消费结构 | 人群规模 / 馆藏数量 / 文献数量 / 传播数据 / 观测数据 / 游客数据 / 教育覆盖 |
| 原理机制 / 工艺流程 / 操作方法 | 学理机制 / 制度机制 / 文本生成机制 / 技艺流程 / 观测方法 / 学习路径 |
| 场景行为 / 使用方式 / 生活实践 | 阅读场景 / 祭祀纪念 / 研究使用 / 公众教育 / 日常实践 / 社群行为 |
| 同类对象事实对照 | 同类人物 / 同类主题 / 同类机构 / 同类科学概念 / 同类文化现象事实对照 |

禁止做法:

- 因为历史人物没有 IPO,就删除“财经资本 / 当代动态 / 最新状态”的资料功能。
- 因为科学主题不是企业,就删除机构网络和最新项目资料。
- 把不适配分支写成“不适用”后停止下钻。
- 在适配分支里写策展启示或展项建议。

## 类型加权

### A 企业 / 机构 / 品牌

使用母骨架,并加重:

- 企业机构 / 品牌主体 / 组织网络
  - 法人主体、总部、成立时间、创始人、官网。
  - 子公司、品牌矩阵、组织结构、领导层。
  - 产品线、业务线、客户、合作伙伴、供应商。
- 数据规模 / 产业市场 / 消费结构
  - 市场规模、行业位置、商业模式、产业链、价格带。
- 财经资本 / 当代动态 / 最新状态
  - 上市/IPO 状态、融资、估值、股权、收入、订单、合同、并购、近一年和近 90 天动态。
- 政策法规 / 制度标准 / 监管伦理
  - 许可、监管文件、诉讼、处罚、合规风险。
- 同类对象事实对照
  - 至少 3 个同类企业或机构,只做事实对照。

### B 博物馆 / 文化馆 / 历史文化

使用母骨架,并加重:

- 历史源流与时间线
  - 生平年谱、对象年表、历史分期、关键事件、当代延续。
- 人物群体与组织网络
  - 家族、师友、同僚、政敌、门生、传承人、研究者、收藏者。
- 地理空间与区域差异
  - 出生地、任职地、流寓地、作品发生地、遗址、纪念地、馆藏地。
- 关键对象 / 载体 / 实物 / 作品 / 档案
  - 作品、文集、书信、奏议、题跋、藏品、文物、照片、影像、碑刻。
- 学术文献 / 权威资料 / 来源出处
  - 正史、地方志、年谱、全集、别集、注本、论文、专著、图录。
- 财经资本 / 当代动态 / 最新状态
  - 转译为当代保护、出版整理、数字化公开、研究会议、纪念活动、教育活动和保护政策。
- 同类对象事实对照
  - 同类人物、同类地域文化、同类文献馆藏、同类保护制度。只比事实,不写可借鉴点。

历史人物类必须有“生平年谱”作为主干,不能只写作品、名句或后世传播。

### C 文旅 / 主题馆 / 主题空间

先把它当**内容主题**研究。主题馆不是先研究运营项目,而是先研究主题本体。

使用母骨架,并加重:

- 基础定义与主题边界
  - 主题是什么,不是什么,与相邻主题区别。
- 历史源流与时间线
  - 主题的起源、分期、传播和当代变化。
- 分类谱系与子主题系统
  - 主题可拆成哪些门类、流派、对象、行为、场景。
- 关键对象 / 载体 / 实物 / 作品 / 档案
  - 主题对应的工具、材料、作品、文本、影像、商品、档案、实物。
- 场景行为 / 使用方式 / 生活实践
  - 参与者如何进入、如何使用、如何进阶、如何形成群体习惯。
- 社会文化 / 符号语言 / 媒介传播
  - 圈层语言、网络传播、身份认同、审美、舆论。
- 数据规模 / 产业市场 / 消费结构
  - 人群规模、市场规模、消费层级、平台数据、赛事/活动数据。
- 政策法规 / 制度标准 / 监管伦理
  - 安全规范、生态伦理、行业制度、争议风险。

若主题绑定具体地点或运营项目,再补资源本底、区位交通、客群市场、运营主体、投资建设、政策规划、生态承载。

### D 其他主题

仍使用母骨架。先判断主题本体属性,再给最相关分支加权:

- 科技/产业:原理机制、分类技术路线、标准专利、产业链、资本动态。
- 人物:生平时间线、作品成果、关系网络、争议、资料出处。
- 城市/地区:地理、历史、人口、产业、文化、政策、最新规划。
- 生活方式/兴趣群体:群体画像、行为流程、工具对象、场景、语言、传播、消费、争议。
- 自然科学/科学教育:直观经验、基础定义、分类体系、科学原理、观测/实验方法、数据参数、科学史、国际组织、任务项目、最新发现。
- 文学艺术/作品专题:作品谱系、版本流传、体裁技法、作者网络、文本/图像证据、馆藏收藏、研究争议、当代传播。

## 认知阶梯骨架

复杂主题必须按认知阶梯展开。它解决的是:小白先能抓住什么,再逐步进入专业内容。

```markdown
- <概念 / 子主题>
  - 直观经验
    - <普通人最先看见、听见、使用或感受到的现象是什么>
  - 基本定义
    - <它是什么>
    - <它不是什么>
  - 边界与误区
    - <最容易混淆的概念>
    - <错误理解为什么错>
  - 分类系统
    - <按什么依据分类>
    - <每一类是什么>
  - 组成与对象
    - <由哪些对象、材料、文本、人物、机构或数据构成>
  - 机制与方法
    - <为什么会这样>
    - <人如何观察、制作、实践、研究或验证它>
  - 场景与行为
    - <谁在什么情况下使用、参与、观看、阅读或研究>
  - 数据与证据
    - <数量、年代、尺寸、参数、规模、文献出处>
  - 历史与前沿
    - <起源与阶段>
    - <最新研究、最新公开、最新政策或最新事件>
  - 来源与争议
    - <权威来源>
    - <不同说法、单一来源、待核实项>
```

使用规则:

- 自然科学、技术、制度、工艺、生活方式主题必须使用认知阶梯。
- 历史人物和文学艺术主题也要使用,但“直观经验”可转为“公众熟悉入口 / 常见印象”。
- 认知阶梯不是可选美化,而是避免一句话定义的基本结构。

## 概念资料卡

遇到任何重要概念,不能只写一句定义,也不能只列几个并列名词。至少覆盖 5 个展开面。

```markdown
- <概念 / 子主题>
  - 定义
    - <用完全不懂的人也能听懂的话解释它是什么>
    - <它不是什么,与相似概念有什么区别>
  - 上下位关系
    - 上位概念
      - <它属于哪个更大的领域,为什么>
    - 下位概念
      - <它下面还能分成哪些类型>
  - 组成结构
    - <组成部分 A>
      - 定义
        - <A 是什么>
      - 作用
        - <A 在整体里负责什么>
      - 来源
        - <证据卡来源标题或 URL>
  - 分类谱系
    - 分类依据
      - <为什么按这个标准分类>
    - <类型 A>
      - 定义
        - <类型 A 是什么>
      - 典型特征
        - <具体特征>
      - 使用/出现条件
        - <在什么场景出现>
      - 代表对象
        - <具体对象和来源>
  - 原理 / 机制 / 流程
    - <第一步 / 第一机制>
      - <具体说明>
    - <第二步 / 第二机制>
      - <具体说明>
  - 场景 / 用途 / 行为
    - <场景 A>
      - <谁在什么情况下使用/参与>
    - <场景 B>
      - <使用目的、限制或风险>
  - 时间线 / 演变
    - <YYYY:发生了什么,来源是什么>
    - <YYYY-MM-DD:发生了什么,来源是什么>
  - 数据 / 参数 / 规模
    - <数值 + 单位 + 时间 + 口径 + 来源>
  - 人物 / 组织 / 代表对象
    - <人物/组织/对象名称>
      - <为什么重要>
      - <对应事件或资料价值>
      - <来源>
  - 故事 / 事件 / 争议
    - <事件名>
      - 时间
        - <具体时间>
      - 地点
        - <具体地点>
      - 人物
        - <相关人物>
      - 经过
        - <具体经过>
      - 结果
        - <具体结果>
      - 争议
        - <不同说法和来源>
  - 来源
    - <URL / 文献名 / 数据库 / 官方文件>
```

- 人群画像与社群结构
- 入门路径与进阶路径
- 工具 / 对象 / 装备 / 材料

- 场景空间与地域差异
- 语言系统 / 黑话 / 媒介传播

## 科学教育骨架

科学主题必须从直观经验进入科学机制,不能直接堆术语。

```markdown
- 公众直观经验
- 基础定义与边界
- 分类体系
- 科学机制 / 物理原理 / 形成过程
- 观测 / 实验 / 验证方法
- 数据参数 / 图谱 / 标准
- 科学史与关键人物
- 科研机构 / 任务网络 / 数据中心
- 最新发现 / 最新任务 / 最新数据发布
- 误区 / 争议 / 不确定性
- 来源索引与待核实清单
```

## 大纲写法规则

- 一级分支使用母骨架。
- 一级分支允许按分支适配规则改名,但必须保留资料功能。
- 二级分支是清晰子主题,不是观点口号。
- 三级以下承载具体内容:定义、事实、时间、数据、来源。
- 每个重要概念至少有 5 个展开面。
- 每个大分支末尾可有“本分支来源”,全图末尾必须有“来源索引与待核实清单”。
- 如果资料冲突,保留冲突版本并标明来源,不要强行合并。
- 如果没有权威来源,写“未找到权威来源”并列出已查方向。

## 反面写法

下面这种写法不合格:

```markdown
- <概念>
  - 定义
    - <一句话解释>
  - 类型
    - <类型 A>  >  <一句话说明>
    - <类型 B>  >  <一句话说明>
  - 需要补充
    - <类型 A> 需要在正式资料中补齐。
```

问题:

- 内容被压缩在同一行。
- 类型没有继续解释。
- 没有组成、原理、场景、数据、时间线、来源。
- 把内部待办写进导图。
- XMind 上只能看到短词,不能让小白读懂。

## 正面写法

下面这种层级才合格:

```markdown
- <概念>
  - 定义
    - <概念是什么>
    - <它与相似概念的区别>
  - 所属领域
    - <上位领域>
      - <为什么属于该领域>
  - 组成结构
    - <组成部分 A>
      - 定义
        - <A 是什么>
      - 作用
        - <A 的作用>
      - 来源
        - <来源标题或 URL>
  - 分类谱系
    - <类型 A>
      - 定义
        - <类型 A 是什么>
      - 工作方式
        - <如何发挥作用>
      - 典型场景
        - <在哪些情况下使用/出现>
      - 局限
        - <常见问题或误区>
    - <类型 B>
      - 定义
        - <类型 B 是什么>
  - 数据与来源
    - <数据事实 + 年份 + 口径 + 来源>
```

## 生成前自检

生成 XMind 前必须运行:

```bash
python3 "<skill>/scripts/outline_audit.py" "06_提炼笔记/调研大纲.md" --type "A/B/C/D" --plan "01_检索计划/research_plan.json"
```

检查:

- 第一分支是否为「主题解读」?
- 是否通过覆盖审计?
- 是否使用统一母骨架,而不是四套割裂目录?
- 是否完成主题类型判定和分支适配?
- C 类主题馆是否先研究主题本体?
- B 类历史人物是否有生平年谱主干?
- 复杂概念是否按认知阶梯展开?
- 每个重要概念是否有 5 个以上展开面?
- 是否仍有 `>` 备注或同一行压缩解释?
- 最大层级是否 ≥7?
- 节点数是否 ≥400,厚重主题是否接近或超过 600?
- 是否存在资料短词叶子?
- 是否有真实项目案例、客户名、私有路径?
- 是否混入策展方案语言?
- 是否混入占位句?
- 是否混入“继续下钻方向 / 关联资料面 / 下钻方式”等方法标签伪内容?

## Word 报告骨架

完整交付模式下,报告 Markdown 建议采用以下结构:

```markdown
# <主题>资料调研报告

## 前言
<说明本报告只做资料整理,与 XMind 共享证据来源。>

## 一、主题总述
<用连续段落讲清主题是什么、资料范围、来源基础。>

## 二、主题边界与基础定义
<定义、别名、上下位关系、易混淆对象。>

## 三、历史源流与发展阶段
<起源、阶段、关键时间线、当代状态。>

## 四、核心对象与分类系统
<产品、作品、器物、工具、文献、机构或子主题。>

## 五、人物、组织与关系网络
<人物身份、组织关系、客户/师友/研究者/监管方。>

## 六、机制、制度或行为逻辑
<技术机制、制度机制、文本机制、实践流程。>

## 七、地理空间与区域差异
<地点、轨迹、设施、遗址、区域比较。>

## 八、数据、市场与当代状态
<规模、收入、用户、馆藏、传播、出版、保护或产业数据。>

## 九、政策、监管、争议与风险
<法规、许可、诉讼、争议、版本差异。>

## 十、同类对象事实对照
<只做事实对照,不写可借鉴点。>

## 来源索引与待核实问题
<来源性质、使用范围、可信度边界、后续更新项。>
```

报告写法:

- 每章先用一段说明本章回答什么。
- 每个小节至少有一段完整段落,不要只有短句。
- 多条证据卡可以合并成一段,但不能丢掉时间、人物、地点、对象、数据和来源口径。
- 章节标题可以按主题适配,但资料功能不能缺失。
- 小节标题要写成正式报告题目,不要用“资料综述”“基本情况”这类空标题。
- 正文要把证据卡字段翻译成自然语言,禁止出现“时间口径为”“相关人物包括”“涉及对象包括”“数据口径包括”“这些信息分别见于”等字段拼接句。
- 来源最好在段末集中说明,写成“以上信息综合参考:来源 A、来源 B。”正文主体先讲事实,不要每句都追着贴来源字段。
