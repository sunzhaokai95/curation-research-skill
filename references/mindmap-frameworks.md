# 知识型思维导图母骨架与概念深挖范式

本 skill 的导图不是调研报告,不是策展方案,也不是证据卡流水账,而是**面向知识传播的资料型 XMind**。

中心主题 = 调研主题。一级分支 = 内容资料维度。每个重要概念都要让完全不懂的人能顺着读懂:先知道它是什么,再知道它不是什么、怎么分类、由什么构成、如何运作、在什么场景出现、有什么数据、有哪些例子、来源如何托底。

## XMind 与 Word 分工

- XMind 保存知识颗粒度,每个概念继续下钻到解释、分类、对象、流程、数据、来源和争议。
- Word 报告组织阅读顺序,把同一批资料转成章节化、段落化、可阅读文本。
- Word 不得替代 XMind 的深度;XMind 不得退化为报告目录。

## 导图写作前置条件

写 `调研大纲.md` 前必须已经完成:

- `research_plan.json`
- `search_results.jsonl` 或真实来源清单
- `sources.jsonl`,包含普通公开网页和人工/浏览器记录的公开摘录
- `evidence_cards.jsonl`
- `evidence_cards.py validate` 通过
- `coverage_audit.py` 通过
- 如果覆盖审计曾失败,已经运行 `research_loop.py` 并完成补搜返工

没有来源和证据卡,不得按骨架空写。骨架是组织资料的容器,不是生成内容的模板。

## 呈现原则

- 默认不使用 `>` 备注;备注数目标为 0。
- 不使用 `概念  >  解释` 这种压缩写法。
- 不把多个解释塞在一个标题里。
- 每个末端资料节点必须是一条完整信息,不能是孤立短词。
- 能继续拆就继续拆,把信息放到下一层。
- 来源和证据要可见,但不能压过知识阅读主线。
- 资料不足时只能有两种写法:继续补搜后写入事实;或写成明确缺口,例如“未找到权威来源:已查 A/B/C,尚缺某对象的时间/地点/来源”。

## 禁止内容

资料型导图中不要出现:

- 展项建议、展示形式、空间表达、展线、互动装置、展厅策划转译。
- 独特价值、核心张力、核心矛盾、主题口号、策展主题。
- 可借鉴点、展陈启示、观众体验设计。
- 真实项目名、客户名、私有文件路径。
- 内部待办句,如“正式使用时要”“需要继续”“需要补”“不能只作为短词存在”。
- 方法标签,如“继续下钻方向”“下钻方式”“关联资料面”“概念资料卡”“来源类型为”“可信度标注为”。
- 字段腔,如“时间口径为”“相关人物包括”“涉及对象包括”“数据口径包括”。

如果脑中出现方案判断,改写成资料事实:时间、参与者、过程、结果、来源和争议。

## 统一内容资料母骨架

A 企业/机构/品牌、B 博物馆/文化馆/历史文化、C 文旅/主题馆/主题空间、D 其他主题,都使用同一套母骨架。类型只改变加权和必查项,不要生成四套互相割裂的目录。

```markdown
# <主题>资料调研
- 主题解读
  - 地域范围
  - 时间范围
  - 主题本体
  - 名称体系
  - 上下位概念
  - 易混淆对象
  - 知识入口来源
  - 权威校准来源
- 基础定义与主题边界
- 历史源流与时间线
- 地理空间与区域差异
- 人物群体与组织网络
- 分类谱系与子主题系统
- 关键对象 / 载体 / 实物 / 作品 / 档案
- 原理机制 / 工艺流程 / 操作方法
- 场景行为 / 使用方式 / 生活实践
- 数据规模 / 产业市场 / 消费结构
- 企业机构 / 品牌主体 / 组织网络
- 财经资本 / 当代动态 / 最新状态
- 政策法规 / 制度标准 / 监管伦理
- 社会文化 / 符号语言 / 媒介传播
- 故事事件 / 争议失败 / 版本差异
- 学术文献 / 权威资料 / 来源出处
- 最新动态与当前状态
- 同类对象事实对照
- 来源索引与待核实清单
```

## 分支适配规则

一级分支承担资料功能,不是固定词语。遇到不适配主题时,允许改名,但必须保留同等资料功能。

| 母骨架分支 | 不适配时的转译方向 |
| --- | --- |
| 企业机构 / 品牌主体 / 组织网络 | 研究机构 / 学术组织 / 主管机构 / 收藏机构 / 社群组织 / 任务网络 |
| 财经资本 / 当代动态 / 最新状态 | 当代保护 / 出版 / 数字化 / 研究进展 / 项目建设 / 公共传播 / 行业动态 |
| 数据规模 / 产业市场 / 消费结构 | 人群规模 / 馆藏数量 / 文献数量 / 传播数据 / 观测数据 / 游客数据 / 教育覆盖 |
| 原理机制 / 工艺流程 / 操作方法 | 学理机制 / 制度机制 / 文本生成机制 / 技艺流程 / 观测方法 / 学习路径 |
| 场景行为 / 使用方式 / 生活实践 | 阅读场景 / 祭祀纪念 / 研究使用 / 公众教育 / 日常实践 / 社群行为 |
| 同类对象事实对照 | 同类人物 / 同类主题 / 同类机构 / 同类科学概念 / 同类文化现象事实对照 |

禁止把不适配分支写成“不适用”后停止下钻。历史人物没有 IPO,也仍要把“财经资本 / 当代动态 / 最新状态”转译为保护、出版、数字化、公共传播和研究动态。

## 类型加权

### A 企业 / 机构 / 品牌

加重主体身份、组织结构、产品业务、技术系统、市场客户、产业链、上市/IPO、融资、估值、收入、订单、监管、诉讼、许可、近一年与近 90 天动态。同类对象至少 3 个,只做事实对照。

### B 博物馆 / 文化馆 / 历史文化

加重生平年谱、对象年表、人物关系、时代制度、地理行旅、作品文献、版本注本、实物馆藏、碑刻图像、地方志、正史、论文专著、当代保护、出版、数字化和纪念活动。历史人物类必须有“生平年谱”作为主干。

### C 文旅 / 主题馆 / 主题空间

先把它当**内容主题**研究。主题馆不是先研究运营项目,而是先研究主题本体:定义、历史、分类、工具对象、行为流程、社群语言、产业消费、政策伦理、安全风险和最新趋势。只有主题绑定具体地点或运营项目时,再补资源本底、区位交通、客群市场、运营主体、投资建设、政策规划和生态承载。

### D 其他主题

仍使用母骨架。先判断主题本体属性,再给最相关分支加权:科技/产业加重原理机制和产业链;人物加重生平时间线和作品成果;城市/地区加重地理、历史、人口、产业、政策;生活方式/兴趣群体加重行为流程、工具对象、场景、语言、传播、消费和争议;自然科学加重直观经验、科学原理、观测方法和最新发现。

## 概念展开结构

遇到任何重要概念,不能只写一句定义,也不能只列几个并列名词。至少覆盖 5 个展开面,厚重主题要覆盖 8 个以上。

```markdown
- <概念 / 子主题>
  - 是什么
    - <用完全不懂的人也能听懂的话解释它是什么>
  - 不是什么
    - <它与相似概念有什么区别>
  - 所属领域
    - <上位概念是什么,为什么属于这里>
  - 分类方式
    - <按什么依据分类>
    - <每一类分别是什么>
  - 组成结构
    - <组成部分 A>
      - <A 是什么>
      - <A 在整体里负责什么>
  - 工作方式 / 形成机制
    - <它为什么会这样运作或形成>
  - 使用场景 / 出现场景
    - <谁在什么情况下使用、参与、观看、阅读或研究>
  - 代表对象 / 人物 / 事件
    - <具体对象、人物或事件>
      - <为什么重要>
      - <时间、地点、来源>
  - 数据 / 参数 / 规模
    - <数值 + 单位 + 时间 + 口径 + 来源>
  - 常见误区 / 争议
    - <误区或不同说法>
  - 证据托底
    - <来源标题、URL、资料性质、采信边界>
```

## 认知阶梯

复杂主题必须按认知阶梯展开:

1. 直观经验:普通人最先看见、听见、使用或感受到什么。
2. 基本定义:它是什么。
3. 边界误区:它不是什么,容易和什么混淆。
4. 分类系统:按什么分,每类是什么。
5. 组成对象:由哪些工具、材料、文本、人物、机构或数据构成。
6. 机制方法:为什么这样,人如何操作、观察、制作、研究或验证。
7. 场景行为:谁在什么情况下参与或使用。
8. 数据证据:数量、年代、尺寸、参数、规模、文献出处。
9. 历史前沿:起源、阶段、最新公开、最新政策或最新事件。
10. 来源争议:权威来源、不同说法、单一来源、待核实项。

```markdown
- 入门路径与进阶路径
- 工具 / 对象 / 装备 / 材料
  - <重要工具或对象>
    - 是什么
    - 不是什么
    - 组成
    - 分类

    - 使用场景
    - 价格或数据

```

### 示例:连杆不能只写成短词

```markdown
- 连杆
  - 是什么
    - 在钓鱼语境中,连杆通常指短时间内连续中鱼或连续上鱼的状态,强调“连续性”和“上鱼结果”。
  - 不是什么
    - 它不是一种鱼竿零件,也不是机械工程里的连接杆;如果资料中出现机械含义,要按语境区分。
  - 与相邻说法的区别
    - 爆护更强调最终鱼获很多,连杆更强调上鱼过程连续。
    - 空军指没有鱼获,与连杆是结果相反的社群表达。
  - 形成条件
    - 鱼情较活跃、窝点聚鱼、饵料适口、线组与钓法匹配时更容易出现。
  - 场景
    - 常见于野钓、黑坑、竞技钓或短视频分享中,用于描述当次作钓节奏。
  - 社群意义
    - 它既是鱼情描述,也是社群里的成就表达,常和“手感”“爆护”“上岸”等词共同出现。
  - 证据托底
    - <钓鱼术语来源、论坛/百科/视频平台资料、发布时间和采信边界>
```

### 示例:硬饵不能只写成“硬质材料制成”

```markdown
- 硬饵
  - 是什么
    - 硬饵是路亚钓法中由塑料、木材、金属等相对坚硬材料制成的拟饵,通过外形、重心、舌板、亮片或拖拽动作模拟小鱼、昆虫、蛙类等猎物。
  - 不是什么
    - 它不同于软饵;软饵主要依靠柔软材质和鱼钩装配产生动作,硬饵通常有固定外形和内置结构。
  - 上位概念
    - 属于路亚拟饵系统,服务于“用假饵诱发鱼类攻击”的钓法逻辑。
  - 主要类型
    - 米诺
      - 通常模拟小鱼游姿,常用于搜索中上层水域。
    - 波趴
      - 在水面拖动时制造声响和水花,常用于激发掠食鱼攻击。
    - VIB
      - 通过震动和反光吸引鱼,常用于快速搜索不同水层。
    - 亮片
      - 依靠金属反光和摆动吸引鱼,结构简单,入门常见。
  - 工作方式
    - 硬饵不是“放进水里就有鱼咬”,而是通过抛投、收线、停顿、抽动和水层控制制造可被鱼感知的目标。
  - 使用场景
    - 与目标鱼种、水深、水流、光照、水体浑浊度和障碍环境有关。
  - 常见误区
    - 只看颜色不看水层和动作,容易把硬饵当成装饰品理解。
  - 证据托底
    - <路亚教材、品牌产品说明、百科/入门资料、钓法文章、发布时间和采信边界>
```

## 科学教育骨架

科学主题必须从直观经验进入科学机制,不能直接堆术语:

```markdown
- 分类体系

- 观测 / 实验 / 验证方法
```

## 大纲写法规则

- 一级分支使用母骨架。
- 一级分支允许按分支适配规则改名,但必须保留资料功能。
- 二级分支是清晰子主题,不是观点口号。
- 三级以下承载具体内容:定义、事实、时间、数据、来源。
- 每个重要概念至少有 5 个展开面。
- 每个大分支末尾可有“本分支来源”,全图末尾必须有“来源索引与待核实清单”。
- 如果资料冲突,保留冲突版本并标明来源,不要强行合并。
- 如果没有权威来源,写“未找到权威来源”并列出已查方向。

## 生成前自检

生成 XMind 前必须运行:

```bash
python3 "<skill>/scripts/outline_audit.py" "06_提炼笔记/调研大纲.md" --type "A/B/C/D" --plan "01_检索计划/research_plan.json"
```

检查:

- 第一分支是否为「主题解读」?
- 是否通过覆盖审计?
- 是否使用统一母骨架,而不是四套割裂目录?
- C 类主题馆是否先研究主题本体?
- B 类历史人物是否有生平年谱主干?
- 复杂概念是否按认知阶梯展开?
- 每个重要概念是否有 5 个以上展开面?
- 是否仍有 `>` 备注或同一行压缩解释?
- 最大层级是否 ≥7?
- 节点数是否 ≥400,厚重主题是否接近或超过 600?
- 是否存在资料短词叶子?
- 是否有真实客户名或私有路径?
- 是否混入策展方案语言?
- 是否混入占位句、字段腔或方法标签伪内容?

## Word 报告骨架

完整交付模式下,报告 Markdown 建议采用以下结构:

```markdown
# <主题>资料调研报告

## 前言
<说明本报告只做资料整理,与 XMind 共享证据来源。>

## 一、主题总述
<用连续段落讲清主题是什么、资料范围、来源基础。>

## 二、主题边界与基础定义
<定义、别名、上下位关系、易混淆对象。>

## 三、历史源流与发展阶段
<起源、阶段、关键时间线、当代状态。>

## 四、核心对象与分类系统
<产品、作品、器物、工具、文献、机构或子主题。>

## 五、人物、组织与关系网络
<人物身份、组织关系、客户/师友/研究者/监管方。>

## 六、机制、制度或行为逻辑
<技术机制、制度机制、文本机制、实践流程。>

## 七、地理空间与区域差异
<地点、轨迹、设施、遗址、区域比较。>

## 八、数据、市场与当代状态
<规模、收入、用户、馆藏、传播、出版、保护或产业数据。>

## 九、政策、监管、争议与风险
```

报告写法:

- 每章先用自然段说明本章回答什么。
- 每个小节至少有一段完整段落,不要只有短句。
- 多条证据卡可以合并成一段,但不能丢掉时间、人物、地点、对象、数据和来源。
- 小节标题要写成正式报告题目,不要用空泛标题。
- 正文要把证据卡字段翻译成自然语言。
- 来源最好在段末集中说明,写成“以上信息综合参考:来源 A、来源 B。”
